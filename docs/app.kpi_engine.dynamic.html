Validates and adjusts the relationship between machines and operations. Ensures the number of machines matches the number of operations. If a single machine string is provided, it resolves to the closest instances and adjusts the operations list if necessary. Handles mismatches by attempting reconciliation or returning an error.

Parameters:

  • machines (str or list) – A string representing a single machine or a list of machine names.

  • operations (list) – A list of operations associated with the machines.

Returns:

A tuple of validated and adjusted machines and operations or an error response.

Return type:

tuple or KPIResponse

Raises:

Exception – If resolving a machine string to its instances fails.

app.kpi_engine.dynamic.dynamic_engine.dynamic_kpi(kpi: str, formulas_dict: dict[str, str], partial_result: dict[str, Any], request: KPIRequest, **kwargs)[source]

Processes a KPI formula string by evaluating and performing operations (D°, A°, S°, R°, C°) and resolving nested expressions. The function replaces parts of the string and recursively calls itself to process nested KPIs, returning the final computed result.

Parameters:

  • kpi (str) – The KPI formula string to be processed. It may contain nested expressions enclosed in square brackets and operations (D°, A°, S°, R°, C°).

  • formulas_dict (dict) – A dictionary containing predefined formulas or other necessary information for the KPI calculation.

  • partial_result (any) – A partial result that is passed to calculation functions. It may contain intermediate data accumulated during processing.

  • request (any) – A request object that may contain additional parameters required for processing, for example in the context of an API or web service.

  • kwargs (dict, optional) – Additional arguments that can be passed to the function as needed.

Raises:

  • KeyError – If an operation in the KPI string does not have a corresponding function in the global scope.

  • ValueError – If the formula contains invalid or unrecognized expressions.

Returns:

The result of the calculated KPI formula, which can be a numeric value or other data type depending on the operation and the processed formula.

Return type:

any

app.kpi_engine.dynamic.dynamic_engine.query_DB(kpi: str, request: KPIRequest, **kwargs) → tuple[ndarray, ndarray][source]

Executes a query on the database to retrieve real-time data based on the provided KPI string, filters, and request parameters. The function processes the query result, organizes the data into a DataFrame, converts it into a NumPy array, and splits it into two parts based on a specified step.

Parameters:

  • kpi (str) – The KPI string used to build the database query. The string is parsed to extract the relevant database field for the query.

  • request (KPIRequest) – An object containing the request parameters, including machines, operations, start and end date, and step for splitting the data.

  • kwargs (dict, optional) – Additional arguments passed to the function as needed.

Raises:

  • ValueError – If the KPI string is not in the expected format or the database reference is invalid.

  • EmptyQueryException – If the query results in an empty dataset (no data is returned from the database).

Returns:

A tuple containing two NumPy arrays: - The first array (step_split) contains the reshaped data split by the specified step. - The second array (bottom) contains any remaining data that could not be split evenly by the step.

Return type:

tuple[np.ndarray, np.ndarray]
