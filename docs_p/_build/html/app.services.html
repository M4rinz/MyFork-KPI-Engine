app.services.data_processing.connect_to_publisher(kpi_streaming_request: KPIStreamingRequest)[source]

Connects to a data producer and starts real-time data processing.

This function sends a POST request to a data preprocessing service, providing a KPIStreamingRequest object as the request body. The function attempts to start the real-time processing of KPI data by invoking the corresponding API endpoint.

Parameters:

kpi_streaming_request (KPIStreamingRequest) – A KPIStreamingRequest object containing the data for the streaming request.

Raises:

requests.exceptions.RequestException – If the request to the data preprocessing service fails.

Returns:

The JSON response from the data preprocessing service or an error message.

Return type:

dict

app.services.database module

app.services.database.insert_aggregated_kpi(request: KPIRequest, kpi_list: list, value: float64)[source]

Inserts the aggregated KPI result into the database.

This function constructs an SQL INSERT query to store the aggregated KPI data into a database. It inserts details such as the KPI name, the aggregated value, the relevant machines and operations, and the list of KPIs involved in the aggregation.

Parameters:

  • request (KPIRequest) – The KPI request containing details such as name, machines, operations, and date range for the aggregation.

  • kpi_list (list) – A list of KPIs that were involved in the aggregation.

  • value (np.float64) – The aggregated KPI value calculated from the input KPIs.

Returns:

The response from the database after insertion.

Return type:

requests.Response

Fetches the closest class instances from the knowledge base.

Fetches the KPI formula from the knowledge base.

This function makes a GET request to a knowledge base service to retrieve the formula for the specified KPI. If the KPI formula is not found, it raises a KPIFormulaNotFoundException.
